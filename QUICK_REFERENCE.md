Copy environment files\ncp backend/.env.example backend/.env\ncp frontend/.env.example frontend/.env.local\n\n# 6. backend/schema.sql\n```\n\n### \"Cannot connect to backend\"\n- Check backend is running: `npm run dev` in backend folder\n- Verify port 5000 is not in use: `lsof -i :5000`\n- Check VITE_API_URL in frontend/.env.local\n\n### \"Stripe payment fails\"\n- Verify STRIPE_PUBLIC_KEY in frontend/.env.local\n- Use test card: 4242 4242 4242 4242\n- Check webhook secret in backend/.env\n\n### \"Email not sending\"\n- Enable Gmail app password: https://myaccount.google.com/apppasswords\n- Use 16-character app password in backend/.env\n- Check EMAIL_USER and EMAIL_PASSWORD are correct\n\n### \"Admin login fails\"\n- Re-run seed script: `npm run seed` in backend folder\n- Check admin email: admin@servicehub.com\n- Check password: admin123\n\n---\n\n## 🔄 API Endpoints Quick Reference\n\n### Auth\n```\nPOST /api/auth/register    {email, password, full_name, phone}\nPOST /api/auth/login       {email, password}\n```\n\n### Services\n```\nGET /api/services                      Get all services\nGET /api/services/:id                  Get specific service\nPOST /api/services                     Create (admin only)\nPUT /api/services/:id                  Update (admin only)\nDELETE /api/services/:id               Delete (admin only)\n```\n\n### Bookings\n```\nPOST /api/bookings/create-session      Create booking + Stripe session\nGET /api/bookings                      Get user's bookings\nPATCH /api/bookings/:id/cancel         Cancel booking\n```\n\n### Admin\n```\nGET /api/admin/bookings                Get all bookings\nPATCH /api/admin/bookings/:id/status   Update booking status\nGET /api/admin/analytics               Get dashboard KPIs\n```\n\n---\n\n## 📊 Database Tables\n\n### users\n- id, email (UNIQUE), password_hash, full_name, phone, created_at\n\n### services\n- id, name, description, price (DECIMAL), duration_minutes, category, image_url, created_at\n\n### bookings\n- id, user_id (FK), service_id (FK), booking_date, booking_time, special_requests, status, stripe_session_id, stripe_payment_id, created_at\n\n### admin_users\n- id, email (UNIQUE), password_hash, full_name, role, created_at\n\n---\n\n## 💡 Pro Tips\n\n1. **Test Everything Locally First** -
